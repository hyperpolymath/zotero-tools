== Phase 1: Foundation

Establish the infrastructure and standards for all Zoterho components.

=== Rhodium Compliance Specification
* [ ] Document Rhodium requirements for Zotero ecosystem tools
* [ ] Define reproducibility criteria (deterministic builds, pinned deps)
* [ ] Establish portability standards (cross-platform, no vendor lock-in)
* [ ] Create compliance verification checklist

=== Build Infrastructure
* [ ] Set up Deno-based build system for ReScript components
* [ ] Configure Rust toolchain with WASM targets
* [ ] Create shared CI/CD templates for all components
* [ ] Implement cross-component dependency management

=== zoterho-template (Baseline)
* [ ] Curate default Zotero 7+ settings for research workflows

* [ ] Include academic citation styles (APA, MLA, Chicago, etc.)
* [ ] Document template customization points

=== Testing Framework
* [ ] Design cross-component integration test suite
* [ ] Set up Zotero headless testing environment

* [ ] Implement snapshot testing for export formats

== Non-Goals

The following are explicitly **out of scope** for Zoterho:

* Replacing Zotero core functionality
* Building a standalone reference manager
* Supporting Zotero versions prior to 7.0
* Mobile applications (use Zotero's official apps)
* Cloud hosting services (self-host or use Zotero sync)

== Versioning Strategy

* **Umbrella repo** (this repo): Calendar versioning (YYYY.MM)
* **Components**: Semantic versioning (MAJOR.MINOR.PATCH)
* **Breaking changes**: Announced one minor version in advance
* **LTS**: None planned; track latest Zotero releases

== Timeline Disclaimer

This roadmap describes *what* will be built, not *when*. Development proceeds as contributor capacity allows. Priorities may shift based on community feedback.

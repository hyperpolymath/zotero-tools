== Overview

Zoterho is a curated ecosystem of tools extending https://www.zotero.org/[Zotero], the open-source reference manager. All components adhere to *Rhodium* compliance standards and the Hyperpolymath language policy (ReScript, Rust, Deno â€” no TypeScript/Node.js).

=== Design Principles

* **Rhodium Compliance** â€” Follows the Rhodium specification for reproducibility and portability
* **FOSS-First** â€” MIT/Palimpsest licensing; no proprietary dependencies
* **Modern Stack** â€” ReScript for type-safe UI logic, Rust for performance-critical paths
* **Multi-Forge** â€” Automatic synchronization to GitHub, GitLab, and Bitbucket

== Quick Start

Clone with submodules:

[source,bash]
----
git clone --recursive https://github.com/hyperpolymath/zoterho.git
cd zoterho
----

Initialize submodules if cloned without `--recursive`:

[source,bash]
----
git submodule update --init --recursive
----

=== Development Setup

This project uses https://github.com/casey/just[just] as its task runner (Makefile is banned per Hyperpolymath policy).

[source,bash]
----
# Install just (if not already installed)
cargo install just

# Setup development environment
just setup

# View all available commands
just --list

# Run policy compliance check
just check-policy
----

== Technology Stack

Per the https://github.com/hyperpolymath[Hyperpolymath] language policy:

[cols="1,2", options="header"]
|===
| Layer | Technology

| Plugin Logic
| ReScript (compiles to JavaScript for Zotero 7+)

| Performance-Critical
| Rust (WASM for browser, native for CLI)

| Runtime
| Deno (replaces Node.js)

| WordPress Plugin
| PHP 8.2+ (exception for WordPress ecosystem)

| Configuration
| Nickel / Guile Scheme

| Package Management
| Guix (primary), Nix (fallback), Deno (JS deps)

|===
